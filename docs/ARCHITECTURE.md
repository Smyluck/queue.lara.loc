# 🏗️ Laravel Horizon - Архитектура и диаграммы

## 📊 Общая архитектура системы

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ВАШЕ ПРИЛОЖЕНИЕ                                 │
│  (Controllers, Commands, API, Events, Listeners)                        │
└────────────────────────────┬────────────────────────────────────────────┘
                             │
                             │ dispatch()
                             │ ProcessEvent::dispatch($data)
                             │
                ┌────────────▼────────────┐
                │   JOB CLASSES           │
                ├────────────────────────┤
                │ • ProcessEvent         │
                │ • SendReport           │
                │ • MonitorQueueLoad     │
                │ • Ваши Job классы      │
                └────────────┬────────────┘
                             │
                             │ Сериализация
                             │ JSON + Metadata
                             │
                ┌────────────▼────────────┐
                │   REDIS QUEUE           │
                ├────────────────────────┤
                │ queues:default          │
                │ queues:high             │
                │ queues:low              │
                │ queues:default:delayed  │
                │ queues:default:reserved │
                └────────────┬────────────┘
                             │
                ┌────────────┴────────────┐
                │                         │
    ┌───────────▼──────────┐  ┌──────────▼──────────┐
    │  HORIZON DASHBOARD   │  │  QUEUE WORKERS      │
    ├──────────────────────┤  ├─────────────────────┤
    │ • Мониторинг         │  │ php artisan horizon │
    │ • Метрики            │  │ (N рабочих)         │
    │ • Пакеты             │  │                     │
    │ • Ожидающие          │  │ Берут задачи из     │
    │ • Выполненные        │  │ очереди и           │
    │ • Отключенные        │  │ выполняют их        │
    │ • Неудачные          │  │                     │
    └──────────────────────┘  └─────────┬───────────┘
                                        │
                                        │ Выполнение
                                        │ handle()
                                        │
                        ┌───────────────▼───────────────┐
                        │      РЕЗУЛЬТАТ                │
                        ├───────────────────────────────┤
                        │ ✅ УСПЕХ                      │
                        │    ↓                          │
                        │    Выполненные задания        │
                        │                               │
                        │ ❌ ОШИБКА                     │
                        │    ↓                          │
                        │    Попытка повтора (backoff)  │
                        │    ↓                          │
                        │    Неудачные задания          │
                        └───────────────────────────────┘
```

## 🔄 Жизненный цикл задачи (детально)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      ЖИЗНЕННЫЙ ЦИКЛ ЗАДАЧИ                              │
└─────────────────────────────────────────────────────────────────────────┘

1. СОЗДАНИЕ (Creation)
   ┌─────────────────────────────────────────┐
   │ ProcessEvent::dispatch($data)           │
   │ ↓                                       │
   │ new ProcessEvent($data)                 │
   │ ↓                                       │
   │ Сериализация в JSON                     │
   └─────────────────────────────────────────┘
                    ↓

2. ОТПРАВКА В ОЧЕРЕДЬ (Queuing)
   ┌─────────────────────────────────────────┐
   │ Redis: LPUSH queues:default {json}      │
   │ ↓                                       │
   │ Horizon видит новую задачу              │
   │ ↓                                       │
   │ Вкладка "Ожидающие выполнения задания"  │
   └─────────────────────────────────────────┘
                    ↓

3. ОЖИДАНИЕ (Waiting)
   ┌─────────────────────────────────────────┐
   │ Задача в очереди                        │
   │ ↓                                       │
   │ Рабочие ищут задачи                     │
   │ ↓                                       │
   │ Horizon показывает статус               │
   └─────────────────────────────────────────┘
                    ↓

4. РЕЗЕРВИРОВАНИЕ (Reserving)
   ┌─────────────────────────────────────────┐
   │ Рабочий берёт задачу                    │
   │ ↓                                       │
   │ Redis: ZSET queues:default:reserved     │
   │ ↓                                       │
   │ Задача зарезервирована                  │
   └─────────────────────────────────────────┘
                    ↓

5. ВЫПОЛНЕНИЕ (Execution)
   ┌─────────────────────────────────────────┐
   │ Десериализация JSON                     │
   │ ↓                                       │
   │ new ProcessEvent($data)                 │
   │ ↓                                       │
   │ $job->handle()                          │
   │ ├─ Log::info()                          │
   │ ├─ Mail::send()                         │
   │ └─ Ваш код                              │
   └─────────────────────────────────────────┘
                    ↓
        ┌───────────┴───────────┐
        │                       │
        ▼                       ▼
   ✅ УСПЕХ              ❌ ОШИБКА
        │                       │
        ▼                       ▼

6a. УСПЕШНОЕ ЗАВЕРШЕНИЕ (Success)
   ┌─────────────────────────────────────────┐
   │ Redis: DEL queues:default:reserved      │
   │ ↓                                       │
   │ Horizon: Выполненные задания            │
   │ ↓                                       │
   │ Логирование успеха                      │
   │ ↓                                       │
   │ КОНЕЦ ✅                                │
   └─────────────────────────────────────────┘

6b. ОШИБКА (Failure)
   ┌─────────────────────────────────────────┐
   │ Exception в handle()                    │
   │ ↓                                       │
   │ Проверка: попытка < tries?              │
   │ ├─ ДА → Повтор                          │
   │ └─ НЕТ → Финальная ошибка               │
   └─────────────────────────────────────────┘
        │
        ├─ ПОВТОР (Retry)
        │  ┌──────────────────────────────┐
        │  │ Задержка (backoff)           │
        │  │ ├─ 1-я попытка: 10 сек       │
        │  │ ├─ 2-я попытка: 30 сек       │
        │  │ └─ 3-я попытка: 60 сек       │
        │  │ ↓                            │
        │  │ Redis: ZADD delayed          │
        │  │ ↓                            │
        │  │ Ждём задержку                │
        │  │ ↓                            │
        │  │ Вернуться на шаг 3           │
        │  └──────────────────────────────┘
        │
        └─ ФИНАЛЬНАЯ ОШИБКА (Failed)
           ┌──────────────────────────────┐
           │ Redis: ZADD failed            │
           │ ↓                            │
           │ Horizon: Неудачные задания   │
           │ ↓                            │
           │ Логирование ошибки           │
           │ ↓                            │
           │ failed() метод               │
           │ ↓                            │
           │ КОНЕЦ ❌                     │
           └──────────────────────────────┘
```

## 🔐 Безопасность

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         БЕЗОПАСНОСТЬ                                    │
└─────────────────────────────────────────────────────────────────────────┘

1. АУТЕНТИФИКАЦИЯ HORIZON
   ├─ config/horizon.php
   │  └─ middleware: ['web', 'auth']
   └─ Только авторизованные пользователи

2. СЕРИАЛИЗАЦИЯ ДАННЫХ
   ├─ SerializesModels trait
   ├─ Безопасная сериализация моделей
   └─ Защита от инъекций

3. ШИФРОВАНИЕ ДАННЫХ
   ├─ Redis AUTH
   ├─ SSL/TLS для Redis
   └─ Шифрование чувствительных данных

4. ЛОГИРОВАНИЕ
   ├─ Все операции логируются
   ├─ Ошибки записываются
   └─ Аудит в storage/logs/

5. ОГРАНИЧЕНИЕ ДОСТУПА
   ├─ Только авторизованные пользователи
   ├─ Проверка прав доступа
   └─ CSRF защита
```

## 📊 Мониторинг и метрики

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    МОНИТОРИНГ И МЕТРИКИ                                 │
└─────────────────────────────────────────────────────────────────────────┘

МЕТРИКИ В HORIZON
├─ Всего обработано
├─ Успешно выполнено
├─ Ошибок
├─ Среднее время выполнения
├─ Пиковая нагрузка
├─ Текущая нагрузка
└─ Графики в реальном времени

ЛОГИРОВАНИЕ
├─ storage/logs/daily-*.log
│  ├─ Успешные задачи
│  ├─ Ошибки
│  ├─ Статистика мониторинга
│  └─ Пороги срабатывания
└─ Уровни: info, warning, error

ПОРОГИ СРАБАТЫВАНИЯ (MonitorQueueLoad)
├─ pending > 100 → ⚠️ WARNING
├─ delayed > 50 → ⚠️ WARNING
└─ total > 150 → 🔴 ERROR

АЛЕРТЫ
├─ Email уведомления
├─ Slack интеграция
├─ Webhook уведомления
└─ Пользовательские обработчики
```
