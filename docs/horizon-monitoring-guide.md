# 📊 Laravel Horizon Monitoring - Полное руководство

## Что такое Horizon Monitoring?

**Laravel Horizon** — это красивый dashboard для мониторинга очередей Redis. Он отслеживает:
- Задачи в очереди
- Выполненные задачи
- Ошибки и сбои
- Производительность рабочих процессов
- Метрики в реальном времени

## 📋 Вкладки Horizon Monitoring

### 1. **Ожидающие выполнения задания** (Pending Jobs)
Показывает задачи, которые ждут обработки в очереди.

**На примере твоего кода:**
```
ProcessEvent #1 - event: "Тестовое событие #1"
ProcessEvent #2 - event: "Тестовое событие #2"
ProcessEvent #3 - event: "Тяжелое тестовое событие #3"
...
```

**Информация:**
- Название задачи (Job class)
- Очередь (queue name)
- Теги для фильтрации
- Время добавления в очередь
- Попытки выполнения

### 2. **Выполненные задания** (Completed Jobs)
Успешно завершённые задачи.

**На примере твоего кода:**
```
✅ ProcessEvent - Выполнена за 245ms
✅ MonitorQueueLoad - Выполнена за 1.2s
```

**Информация:**
- Время выполнения
- Дата завершения
- Результат (успех/ошибка)

### 3. **Отключенные задания** (Paused Jobs)
Задачи, которые были вручную отключены.

### 4. **Неудачные задания** (Failed Jobs)
Задачи, которые упали с ошибками.

**На примере твоего кода:**
```
❌ ProcessEvent - Ошибка: "SMTP connection failed"
❌ MonitorQueueLoad - Ошибка: "Redis connection timeout"
```

**Информация:**
- Сообщение об ошибке
- Stack trace
- Попытки повтора
- Возможность переотправки

## 🏷️ Теги (Tags) - Ключевая фишка

Теги позволяют фильтровать и группировать задачи в Horizon.

### Как работают теги в твоем коде:

#### ProcessEvent с тегами:
```php
public function tags(): array
{
    return [
        'event:' . ($this->data['event'] ?? 'unknown'),
        'user:' . ($this->data['data']['user_id'] ?? 'unknown'),
        'type:' . ($this->data['data']['metadata']['type'] ?? 'unknown'),
    ];
}
```

**Пример тегов для задачи:**
```
event:Тестовое событие #1
user:42
type:normal
```

#### MonitorQueueLoad с тегами:
```php
public function tags(): array
{
    return [
        'monitoring',
        'queue-stats',
        'critical',
    ];
}
```

**Пример тегов:**
```
monitoring
queue-stats
critical
```

### Использование тегов в Horizon:
1. Откройте вкладку **"Мониторинг"**
2. Нажмите на тег (например, `user:42`)
3. Увидите все задачи этого пользователя
4. Можете отследить историю его задач

## 🔄 Как работает твой мониторинг

### Шаг 1: Генерация нагрузки
```bash
php artisan queue:generate-load 200 --type=mixed
```

Команда создаёт 200 задач `ProcessEvent`:
- 100 обычных (normal)
- 100 тяжёлых (heavy)

### Шаг 2: Задачи попадают в очередь Redis
```
Redis Queue: default
├── ProcessEvent #1 (normal)
├── ProcessEvent #2 (heavy)
├── ProcessEvent #3 (normal)
└── ... 197 ещё задач
```

### Шаг 3: Horizon отслеживает задачи
- Видит новые задачи в очереди
- Отображает их в вкладке "Ожидающие выполнения задания"
- Показывает теги для фильтрации

### Шаг 4: Рабочие процессы выполняют задачи
```bash
php artisan horizon
```

Horizon запускает рабочих, которые:
1. Берут задачу из очереди
2. Выполняют `handle()` метод
3. Отправляют email через `Mail::to()`
4. Логируют результат
5. Перемещают задачу в "Выполненные"

### Шаг 5: Мониторинг каждые 5 минут
```
schedule.php:
$schedule->job(new MonitorQueueLoad())
    ->everyFiveMinutes()
```

`MonitorQueueLoad` проверяет:
- Количество ожидающих задач
- Количество отложенных задач
- Количество зарезервированных задач
- Срабатывает ли пороги (100+ pending = ⚠️)

## 🚀 Как использовать Horizon

### 1. Запустить Horizon
```bash
php artisan horizon
```

### 2. Открыть Dashboard
```
http://localhost/horizon
```

### 3. Генерировать нагрузку
```bash
php artisan queue:generate-load 50 --type=normal
php artisan queue:generate-load 50 --type=heavy
php artisan queue:generate-load 100 --type=mixed
```

### 4. Мониторить в реальном времени
- Вкладка "Ожидающие выполнения задания" - видите новые задачи
- Вкладка "Выполненные задания" - видите завершённые
- Вкладка "Неудачные задания" - видите ошибки
- Вкладка "Метрики" - видите статистику

### 5. Фильтровать по тегам
- Нажмите на тег в списке задач
- Увидите только задачи с этим тегом
- Можете отследить конкретного пользователя или тип события

## 🔧 Конфигурация Horizon

### config/horizon.php

```php
// Пороги для уведомлений о задержках
'waits' => [
    'redis:default' => 60,  // Если задача ждёт >60 сек
],

// Балансировщик нагрузки
'balance' => 'simple',  // или 'auto'

// Максимум попыток
'max_processes' => 1,

// Минимум рабочих
'min_processes' => 1,
```

## 📝 Логирование в Horizon

Твой код логирует в разные каналы:

```php
// ProcessEvent логирует обработку
Log::info("Обработка события: {$this->data['event']}", $this->data);

// MonitorQueueLoad логирует статистику
Log::channel('daily')->info('Queue Load Monitor', $queueStats);

// Пороги срабатывают с warning/error
Log::warning("⚠️ ВНИМАНИЕ: В очереди {$stats['pending']} ожидающих задач");
Log::error("🔴 КРИТИЧНО: Всего {$stats['total']} задач в очереди");
```

## 🎓 Практические советы

### 1. Отслеживание конкретного пользователя
```php
// Нажмите на тег user:42 в Horizon
// Увидите все задачи пользователя 42
```

### 2. Поиск ошибок
```
Вкладка "Неудачные задания" → Нажмите на задачу → Увидите stack trace
```

### 3. Переотправка задачи
```
Вкладка "Неудачные задания" → Нажмите "Retry" → Задача вернётся в очередь
```

### 4. Остановка задачи
```
Вкладка "Ожидающие выполнения задания" → Нажмите "Pause" → Задача не будет выполнена
```

### 5. Анализ производительности
```
Вкладка "Метрики" → Смотрите среднее время выполнения
Если >5s → оптимизируйте handle() метод
```

## 🐛 Частые проблемы

### Проблема: "Вы не отслеживаете теги"
**Решение:** Добавьте метод `tags()` в Job класс (уже сделано в твоём коде)

### Проблема: Задачи не выполняются
**Решение:** Убедитесь, что запущен `php artisan horizon`

### Проблема: Horizon не обновляется
**Решение:** Проверьте Redis соединение в `.env`

### Проблема: Много ошибок в "Неудачные задания"
**Решение:** Проверьте логи в `storage/logs/`
