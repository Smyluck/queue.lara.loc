# Техническое задание: Интеграция RabbitMQ в Laravel 12

✅ **Задание**: Реализация сервиса для работы с очередями на базе RabbitMQ.

---

## 📌 Цель
Реализовать простой сервис, который:
1. **Генерирует события** (публикует сообщения в очередь).
2. **Обрабатывает сообщения** (консюмер читает и выполняет работу).
3. **Имеет автоматические тесты** для проверки отправки, приёма и обработки сообщений.

---

## 🔧 Технические требования

### **Технологии**
- **PHP**: ≥ 8.0
- **Laravel**: 12.x
- **RabbitMQ**: Локально через Docker или на хосте.
- **Клиент для RabbitMQ**: `vladimir-yuldashev/laravel-queue-rabbitmq` или `php-amqplib`.

### **Компоненты**

#### **1. Producer (Отправитель сообщений)**
- Класс или контроллер, который отправляет JSON-сообщение в очередь.
- Пример сообщения:
  ```json
  {
    "event": "user_registered",
    "data": {
      "user_id": 1,
      "email": "user@example.com"
    }
  }

#### **2. Worker/Consumer (Обработчик сообщений)**
Класс, который получает сообщение из очереди и выполняет действие:

- Сохраняет запись в базе данных.
- Имитирует отправку письма.
- Пишет в лог.

#### **3. Настройка очереди**

- Очередь на базе RabbitMQ.
- Параметры (имя, обменник, тип) на усмотрение разработчика.

#### **4. Тесты**

Юнит/интеграционные тесты для проверки:

- Отправки сообщений.
- Обработки сообщений консюмером.

Использовать PHPUnit + Mocks или реальный RabbitMQ в Docker-контейнере.

#### **5. Документация**

README.md с инструкциями:

- Установка зависимостей (composer install).
- Запуск RabbitMQ (docker-compose up -d rabbitmq).
- Команды для запуска Producer и Worker.
- Запуск тестов (php artisan test).


#### **6. Возможные расширения (опционально)**

- Отложенные задачи: Отправка сообщений с задержкой.
- Retry/Dead Letter Queue: Повторная обработка при ошибке.
- Метрики: Счётчик успешных/неудачных сообщений.
- REST-API: Эндпоинт для генерации событий.

#### **7. Ожидаемый результат**

- Репозиторий с рабочим кодом.
- Рабочий проект, который можно запустить и продемонстрировать.
- Тесты, которые проходят командой php artisan test.
- README.md с инструкциями по запуску.
