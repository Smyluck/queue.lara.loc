# 🚀 Laravel Horizon - Практические примеры использования

## Быстрый старт

### 1. Запустить Horizon
```bash
php artisan horizon
```

### 2. Открыть Dashboard
```
http://localhost/horizon
```

### 3. Отправить тестовые задачи
```bash
# Вариант 1: Через API
curl -X POST http://localhost/api/horizon/demo/single-event

# Вариант 2: Через команду
php artisan queue:generate-load 50 --type=normal
```

## 🎯 Пример 1: Одно событие

### Отправить одно событие
```bash
curl -X POST http://localhost/api/horizon/demo/single-event
```

### Что происходит:
1. **Создаётся задача** `ProcessEvent` с данными:
   ```php
   [
       'event' => 'Demo Event',
       'data' => [
           'user_id' => 123,
           'email' => 'demo@example.com',
       ]
   ]
   ```

2. **Задача попадает в очередь Redis**

3. **В Horizon Dashboard видите:**
   - Вкладка "Ожидающие выполнения задания" → 1 новая задача
   - Теги: `event:Demo Event`, `user:123`, `type:normal`

4. **После запуска рабочих** (`php artisan horizon`):
   - Задача выполняется
   - Email отправляется
   - Задача переходит в "Выполненные задания"

### Как отследить в Horizon:
```
1. Откройте http://localhost/horizon
2. Вкладка "Мониторинг" → видите теги
3. Нажмите на тег "event:Demo Event"
4. Увидите только эту задачу
5. Нажмите на задачу → увидите детали
```

## 📦 Пример 2: Пакет событий

### Отправить 10 событий
```bash
curl -X POST "http://localhost/api/horizon/demo/batch-events?count=10"
```

### Отправить 50 событий
```bash
curl -X POST "http://localhost/api/horizon/demo/batch-events?count=50"
```

### Что происходит:
1. **Создаётся 10 задач** `ProcessEvent`
2. **Каждая задача имеет разного пользователя** (user_id от 1 до 10)
3. **В Horizon видите:**
   - Вкладка "Ожидающие выполнения задания" → 10 новых задач
   - Теги: `user:1`, `user:2`, ..., `user:10`

### Фильтрация по пользователю:
```
1. Откройте http://localhost/horizon
2. Вкладка "Мониторинг"
3. Нажмите на тег "user:5"
4. Увидите только задачи пользователя 5
5. Можете отследить все его события
```

### Мониторинг в реальном времени:
```
1. Откройте Horizon в одном окне
2. В другом окне запустите: php artisan horizon
3. Смотрите как задачи исчезают из "Ожидающие"
4. Появляются в "Выполненные"
5. Проверьте "Метрики" → видите статистику
```

## 📧 Пример 3: Отправка отчётов

### Отправить daily отчёты
```bash
curl -X POST "http://localhost/api/horizon/demo/send-reports?type=daily"
```

### Отправить weekly отчёты
```bash
curl -X POST "http://localhost/api/horizon/demo/send-reports?type=weekly"
```

### Что происходит:
1. **Создаётся 3 задачи** `SendReport`:
   - admin@example.com
   - manager@example.com
   - support@example.com

2. **Каждая задача имеет:**
   - 3 попытки выполнения (tries=3)
   - Задержка между попытками: 10s, 30s, 60s
   - Теги: `report:daily`, `email:admin`, `mailing`

3. **В Horizon видите:**
   - Вкладка "Ожидающие выполнения задания" → 3 SendReport задачи
   - Теги для фильтрации

### Отследить отправку отчётов:
```
1. Откройте http://localhost/horizon
2. Вкладка "Мониторинг" → нажмите "report:daily"
3. Видите все daily отчёты
4. Нажмите на конкретный отчёт → детали
5. Если ошибка → видите попытки повтора
```

### Если отчёт упал:
```
1. Вкладка "Неудачные задания"
2. Найдите SendReport с ошибкой
3. Нажмите "Retry" → задача вернётся в очередь
4. Будет ещё 2 попытки
```

## 🔍 Пример 4: Мониторинг вручную

### Запустить мониторинг
```bash
curl -X POST http://localhost/api/horizon/demo/monitor-now
```

### Что происходит:
1. **Создаётся задача** `MonitorQueueLoad`
2. **Задача проверяет:**
   - Количество ожидающих задач
   - Количество отложенных задач
   - Количество зарезервированных задач

3. **Если пороги превышены:**
   - pending > 100 → ⚠️ WARNING
   - total > 150 → 🔴 ERROR

4. **В Horizon видите:**
   - Вкладка "Ожидающие выполнения задания" → MonitorQueueLoad
   - Теги: `monitoring`, `queue-stats`, `critical`

### Проверить логи мониторинга:
```bash
tail -f storage/logs/daily-*.log | grep "Queue Load Monitor"
```

## 🎬 Пример 5: Комплексный сценарий

### Запустить полный сценарий
```bash
curl -X POST http://localhost/api/horizon/demo/complex-scenario
```

### Что происходит:
1. **Создаётся 20 событий** `ProcessEvent`
   - 10 обычных (normal)
   - 10 тяжёлых (heavy)
   - Разные пользователи (user_id от 1 до 5)

2. **Создаётся 1 отчёт** `SendReport`
   - Тип: complex_demo
   - Email: admin@example.com

3. **Запускается мониторинг** `MonitorQueueLoad`

4. **Всего 22 задачи в очереди**

### Полный цикл мониторинга:
```
Время 0:00 - Запускаем сценарий
  Ожидающие: 22
  Выполненные: 0

Время 0:05 - Рабочие обработали часть
  Ожидающие: 15
  Выполненные: 7

Время 0:10 - Мониторинг срабатывает
  Логирует: pending=15, delayed=0, reserved=7

Время 0:15 - Все обработано
  Ожидающие: 0
  Выполненные: 22
  Среднее время: 1.2s
```

### Анализ в Horizon:
```
1. Вкладка "Метрики" → видите графики
2. Вкладка "Мониторинг" → фильтруйте по тегам
3. Вкладка "Выполненные задания" → видите все 22 задачи
4. Нажмите на user:3 → видите только его события
5. Нажмите на report:complex_demo → видите отчёт
```

## 🔧 Отладка и решение проблем

### Проблема: Задачи не выполняются

**Решение:**
```bash
# 1. Проверьте, запущен ли Horizon
ps aux | grep horizon

# 2. Если нет, запустите
php artisan horizon

# 3. Проверьте Redis соединение
redis-cli ping
# Должно вернуть: PONG

# 4. Проверьте логи
tail -f storage/logs/daily-*.log
```

### Проблема: Много ошибок в "Неудачные задания"

**Решение:**
```bash
# 1. Откройте Horizon
# 2. Вкладка "Неудачные задания"
# 3. Нажмите на задачу с ошибкой
# 4. Смотрите stack trace
# 5. Исправьте ошибку в коде
# 6. Нажмите "Retry" для переотправки
```

### Проблема: Horizon не обновляется

**Решение:**
```bash
# 1. Проверьте Redis
redis-cli
> KEYS horizon:*
# Должны быть ключи

# 2. Очистите Redis кэш
redis-cli FLUSHDB

# 3. Перезагрузите Horizon
# Ctrl+C в терминале
php artisan horizon
```

### Проблема: "Вы не отслеживаете теги"

**Решение:**
```php
// Добавьте метод tags() в Job класс
public function tags(): array
{
    return [
        'event:' . $this->data['event'],
        'user:' . $this->data['user_id'],
    ];
}
```

## 📈 Оптимизация производительности

### 1. Увеличить количество рабочих
```php
// config/horizon.php
'environments' => [
    'production' => [
        'supervisor-1' => [
            'maxProcesses' => 10,  // Было 1
            'minProcesses' => 5,   // Было 1
        ],
    ],
],
```

### 2. Оптимизировать handle() метод
```php
// ❌ Плохо - долгая операция
public function handle(): void
{
    sleep(10);  // 10 секунд!
    Mail::send(...);
}

// ✅ Хорошо - быстрая операция
public function handle(): void
{
    Mail::queue(...);  // Отправляем в фоне
}
```

### 3. Использовать разные очереди
```php
// config/horizon.php
'queues' => [
    'default' => ['balance' => 'simple', 'maxProcesses' => 1],
    'high' => ['balance' => 'simple', 'maxProcesses' => 5],
    'low' => ['balance' => 'simple', 'maxProcesses' => 1],
],

// В коде
ProcessEvent::dispatch($data)->onQueue('high');  // Приоритет
```

```
└────────────────────┬────────────────────────────────────┘
                     │

        ┌────────────────────────┐
        │  Dispatch Job          │
        │  ProcessEvent::dispatch│

                     │
                     ▼
        ┌────────────────────────┐
        │  Redis Queue           │
        │  (очередь задач)       │
        └────────────┬───────────┘
                     │
        ┌────────────┴────────────┐
        │                         │
        ▼                         ▼
    ┌────────────┐          ┌──────────────┐
    │  Horizon   │          │  Рабочие     │
    │  Dashboard │          │  (Workers)   │
    │  (UI)      │          │  php artisan │
    │            │          │  horizon     │
    └────────────┘          └──────┬───────┘
                                   │
                                   ▼
                        ┌──────────────────┐
                        │  Выполнение      │
                        │  handle() метода │
                        │  Отправка email  │
                        │  Логирование     │
                        └──────────────────┘
```

## 📚 Полезные команды

```bash
# Запустить Horizon
php artisan horizon

# Остановить Horizon
php artisan horizon:terminate

# Перезагрузить Horizon
php artisan horizon:restart

# Сделать снимок метрик
php artisan horizon:snapshot

# Очистить неудачные задачи
php artisan horizon:forget-failed

# Генерировать нагрузку
php artisan queue:generate-load 100 --type=mixed

# Запустить мониторинг вручную
php artisan queue:monitor

# Проверить статус очереди
php artisan queue:failed

# Переотправить неудачную задачу
php artisan queue:retry {id}
```
