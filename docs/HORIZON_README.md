## Что такое Horizon Monitoring?

**Laravel Horizon** — это красивый веб-интерфейс для мониторинга очередей Redis в реальном времени.

### Основные возможности:
- ✅ Видеть все задачи в очереди
- ✅ Отслеживать выполнение в реальном времени
- ✅ Фильтровать по тегам
- ✅ Анализировать метрики и производительность
- ✅ Переотправлять неудачные задачи
- ✅ Мониторить нагрузку на систему

### Почему это важно?
```
Без Horizon:
❌ Не видите что происходит с задачами
❌ Сложно отладить ошибки
❌ Нет статистики производительности

С Horizon:
✅ Видите всё в реальном времени
✅ Быстро находите и исправляете ошибки
✅ Анализируете производительность
```

## Быстрый старт

### Шаг 1: Запустить Horizon
```bash
php artisan horizon
```

Вы должны увидеть:
```
Starting Horizon...
Horizon started successfully.
```

### Шаг 2: Открыть Dashboard
```
http://localhost/horizon
```

### Шаг 3: Отправить задачи

**Вариант A: Через API**
```bash
curl -X POST http://localhost/api/horizon/demo/single-event
```

**Вариант B: Через команду**
```bash
php artisan queue:generate-load 50 --type=normal
```

**Вариант C: Через контроллер**
```php
ProcessEvent::dispatch($data)->onQueue('default');
```

### Шаг 4: Смотреть результаты

В Horizon Dashboard вы увидите:
- Вкладка "Ожидающие выполнения задания" → новые задачи
- Вкладка "Выполненные задания" → завершённые задачи
- Вкладка "Метрики" → статистика

## Вкладки Horizon

### 1️⃣ Мониторинг (Monitoring)

**Что показывает:**
- Список всех тегов
- Количество задач с каждым тегом
- Быстрый поиск по тегам

**Пример:**
```
event:Demo Event          (1 задача)
user:1                    (5 задач)
user:2                    (3 задачи)
type:normal               (7 задач)
type:heavy                (1 задача)
report:daily              (1 задача)
monitoring                (1 задача)
```

**Как использовать:**
1. Нажмите на тег (например, `user:1`)
2. Увидите только задачи этого пользователя
3. Можете отследить историю

---

### 2️⃣ Метрики (Metrics)

**Что показывает:**
- Графики производительности
- Статистика обработки
- Пиковая нагрузка
- Среднее время выполнения

**Пример:**
```
┌─────────────────────────────────────────┐
│ Всего обработано          1,245         │
│ Успешно выполнено         1,200         │
│ Ошибок                    45            │
│ Среднее время             1.2s          │
│ Пиковая нагрузка          156 задач     │
│ Текущая нагрузка          12 задач      │
└─────────────────────────────────────────┘
```

**Как использовать:**
1. Смотрите графики в реальном времени
2. Анализируйте тренды
3. Выявляйте проблемы рано

---

### 3️⃣ Пакеты (Batches)

**Что показывает:**
- Батчи задач (группы)
- Прогресс выполнения
- Статус каждого батча

**Как использовать:**
1. Отправьте пакет задач
2. Смотрите прогресс
3. Видите когда всё завершится

---

### 4️⃣ Ожидающие выполнения задания (Pending Jobs)

**Что показывает:**
- Задачи в очереди
- Информация о каждой задаче
- Теги для фильтрации

**Пример:**
```
┌──────────────────────────────────────────────────┐
│ Задача              Очередь    Попытки  Теги    │
├──────────────────────────────────────────────────┤
│ ProcessEvent        default    0/3      user:1  │
│ ProcessEvent        default    0/3      user:2  │
│ SendReport          default    0/3      email:* │
│ MonitorQueueLoad    default    0/1      monitor │
└──────────────────────────────────────────────────┘
```

**Как использовать:**
1. Видите новые задачи
2. Нажмите на задачу → детали
3. Нажмите "Pause" → остановить
4. Нажмите "Delete" → удалить

---

### 5️⃣ Выполненные задания (Completed Jobs)

**Что показывает:**
- Успешно завершённые задачи
- Время выполнения
- Дата завершения

**Пример:**
```
┌──────────────────────────────────────────────────┐
│ Задача              Время      Статус   Дата    │
├──────────────────────────────────────────────────┤
│ ProcessEvent        245ms      ✅       16:45   │
│ ProcessEvent        312ms      ✅       16:45   │
│ SendReport          1.2s       ✅       16:46   │
│ MonitorQueueLoad    450ms      ✅       16:46   │
└──────────────────────────────────────────────────┘
```

**Как использовать:**
1. Проверьте что задачи выполнились
2. Нажмите на задачу → логи
3. Видите время выполнения

---

### 6️⃣ Отключенные задания (Paused Jobs)

**Что показывает:**
- Паузированные задачи
- Причина паузы

**Как использовать:**
1. Нажмите "Resume" → возобновить
2. Задача вернётся в очередь

---

### 7️⃣ Неудачные задания (Failed Jobs)

**Что показывает:**
- Задачи с ошибками
- Сообщение об ошибке
- Stack trace
- Количество попыток

**Пример:**
```
┌──────────────────────────────────────────────────┐
│ Задача              Ошибка              Попытки │
├──────────────────────────────────────────────────┤
│ SendReport          SMTP timeout         1/3    │
│ ProcessEvent        Email invalid        0/3    │
└──────────────────────────────────────────────────┘
```

**Как использовать:**
1. Нажмите на задачу → детали
2. Смотрите stack trace
3. Исправьте ошибку в коде
4. Нажмите "Retry" → переотправить
5. Задача вернётся в очередь

## Теги и фильтрация

### Что такое теги?

Теги — это метки для задач, которые позволяют быстро их найти и отфильтровать.

### Как добавить теги?

```php
class ProcessEvent implements ShouldQueue
{
    public function tags(): array
    {
        return [
            'event:' . $this->data['event'],
            'user:' . $this->data['user_id'],
            'type:' . $this->data['type'],
        ];
    }
}
```

### Примеры тегов:

```
event:Demo Event          ← Тип события
user:42                   ← ID пользователя
type:normal               ← Тип задачи
report:daily              ← Тип отчёта
email:admin               ← Email адрес
monitoring                ← Категория
queue-stats               ← Подкатегория
critical                  ← Приоритет
```

### Как фильтровать?

```
1. Откройте http://localhost/horizon
2. Вкладка "Мониторинг"
3. Нажмите на тег (например, user:42)
4. Видите только задачи этого пользователя
5. Нажмите на задачу → детали
```

### Практические примеры фильтрации:

**Найти все задачи пользователя 42:**
```
Нажмите на тег: user:42
```

**Найти все ошибки отправки email:**
```
Вкладка "Неудачные задания"
Нажмите на тег: email:*
```

**Найти все критические задачи:**
```
Нажмите на тег: critical
```

## Примеры использования

### Пример 1: Обработка одного события

```bash
curl -X POST http://localhost/api/horizon/demo/single-event
```

**Результат:**
- 1 задача `ProcessEvent` в очереди
- Теги: `event:Demo Event`, `user:123`, `type:normal`
- После выполнения → email отправляется

### Пример 2: Пакет событий

```bash
curl -X POST "http://localhost/api/horizon/demo/batch-events?count=10"
```

**Результат:**
- 10 задач `ProcessEvent` в очереди
- Разные пользователи (user_id от 1 до 10)
- Теги: `user:1`, `user:2`, ..., `user:10`

### Пример 3: Отправка отчётов

```bash
curl -X POST "http://localhost/api/horizon/demo/send-reports?type=daily"
```

**Результат:**
- 3 задачи `SendReport` в очереди
- Для разных адресов: admin, manager, support
- Каждая имеет 3 попытки выполнения
- Теги: `report:daily`, `email:admin`, `mailing`

### Пример 4: Мониторинг вручную

```bash
curl -X POST http://localhost/api/horizon/demo/monitor-now
```

**Результат:**
- 1 задача `MonitorQueueLoad` в очереди
- Проверяет пороги нагрузки
- Логирует статистику
- Теги: `monitoring`, `queue-stats`, `critical`

### Пример 5: Комплексный сценарий

```bash
curl -X POST http://localhost/api/horizon/demo/complex-scenario
```

**Результат:**
- 20 событий `ProcessEvent` (normal + heavy)
- 1 отчёт `SendReport`
- 1 мониторинг `MonitorQueueLoad`
- Всего 22 задачи в очереди

## Отладка и решение проблем

### Проблема: Задачи не выполняются

**Решение:**
```bash
# 1. Проверьте Horizon запущен
ps aux | grep horizon

# 2. Если нет, запустите
php artisan horizon

# 3. Проверьте Redis
redis-cli ping
# Должно вернуть: PONG

# 4. Проверьте логи
tail -f storage/logs/daily-*.log
```

### Проблема: "Вы не отслеживаете теги"

**Решение:**
```php
// Добавьте метод tags() в Job класс
public function tags(): array
{
    return [
        'event:' . $this->data['event'],
        'user:' . $this->data['user_id'],
    ];
}
```

### Проблема: Много ошибок в "Неудачные задания"

**Решение:**
```
1. Откройте Horizon
2. Вкладка "Неудачные задания"
3. Нажмите на задачу с ошибкой
4. Смотрите stack trace
5. Исправьте ошибку в коде
6. Нажмите "Retry" для переотправки
```

### Проблема: Horizon не обновляется

**Решение:**
```bash
# 1. Проверьте Redis
redis-cli
> KEYS horizon:*
# Должны быть ключи

# 2. Очистите Redis кэш
redis-cli FLUSHDB

# 3. Перезагрузите Horizon
php artisan horizon:restart
```

### Проблема: Задачи выполняются очень медленно

**Решение:**
```php
// config/horizon.php
'maxProcesses' => 10,  // Увеличьте количество рабочих
'minProcesses' => 5,
```

## 💡 Полезные советы

1. **Всегда добавляйте теги** — они помогают быстро найти задачи
2. **Используйте разные очереди** — для приоритизации
3. **Логируйте всё** — помогает при отладке
4. **Проверяйте пороги** — настройте уведомления
5. **Мониторьте метрики** — видите проблемы рано
6. **Переотправляйте ошибки** — не теряйте данные
7. **Оптимизируйте handle()** — быстрее = лучше
8. **Используйте backoff** — умные повторы при ошибках
